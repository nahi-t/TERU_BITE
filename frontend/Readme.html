2. Database Architecture (MVP Schema)

The database schema is structured for transactional safety, quick relational indexing, and scaling. It handles category categorization, multiple pricing tiers per item (variants), and ingredient management out of the box.

Table: categories Stores main menu classifications

Column Name Type Constraints Description
idUUIDPRIMARY KEYUnique category identifier.
nameVARCHAR(100)NOT NULLDisplay name (e.g., "Burgers").
slugVARCHAR(100)UNIQUE, NOT NULLURL-friendly slug (e.g., "burgers").
display_orderINTDEFAULT 0Controls tab sequence position.
is_activeBOOLEANDEFAULT TRUEToggle status of the whole category.

Table: menu_items Main menu catalog

Column Name Type Constraints Description
idUUIDPRIMARY KEYUnique item identifier.
category_idUUIDREFERENCES categories(id)Associated category.
nameVARCHAR(150)NOT NULLDisplay name (e.g., "The Wow Signature").
short_descriptionVARCHAR(255)NOT NULLBrief taglines shown in preview grids.
descriptionTEXTNOT NULLDetailed description for item view.
image_urlVARCHAR(512)NULLDynamic hosting image link.
is_availableBOOLEANDEFAULT TRUEShows/hides item based on daily stock.

Table: item_variants Handles flexible, multi-tier pricing structures

Column Name Type Constraints Description
idUUIDPRIMARY KEYUnique variant identifier.
menu_item_idUUIDREFERENCES menu_items(id)Associated base menu item.
nameVARCHAR(100)NOT NULLVariant identifier (e.g., "Double", "Regular").
priceDECIMAL(10,2)NOT NULLThe specific variant price.
is_availableBOOLEANDEFAULT TRUEAllows shutting down specific sizing lines.

3. SQL DDL Implementation

Below is standard SQL (PostgreSQL compatible) used to construct and execute the backend database architecture: 

CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

CREATE TABLE categories (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    name VARCHAR(100) NOT NULL,
    slug VARCHAR(100) UNIQUE NOT NULL,
    display_order INT NOT NULL DEFAULT 0,
    is_active BOOLEAN NOT NULL DEFAULT TRUE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE menu_items (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    category_id UUID NOT NULL REFERENCES categories(id) ON DELETE RESTRICT,
    name VARCHAR(150) NOT NULL,
    slug VARCHAR(150) UNIQUE NOT NULL,
    short_description VARCHAR(255) NOT NULL,
    description TEXT NOT NULL,
    image_url VARCHAR(512),
    is_available BOOLEAN NOT NULL DEFAULT TRUE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE item_variants (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    menu_item_id UUID NOT NULL REFERENCES menu_items(id) ON DELETE CASCADE,
    name VARCHAR(100) NOT NULL,
    price DECIMAL(10, 2) NOT NULL CHECK (price >= 0),
    is_available BOOLEAN NOT NULL DEFAULT TRUE
);
